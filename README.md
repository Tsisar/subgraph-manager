Збереження Ключа у SSH Агенті (необов'язково):

Якщо ви часто підключаєтеся до контейнера, ви можете додати ваш приватний ключ до SSH агента, щоб не вводити його шлях кожного разу. Для цього виконайте:
```shell
ssh-add ./docker/ssh-keys/docker_ssh_key
```
Після цього ви можете підключатися до контейнера без вказівки шляху до ключа:
```shell
ssh root@localhost -p 2222
```

```shell
ssh -i ./docker/ssh-keys/docker_ssh_key root@localhost -p 2222
```
Переконайтеся, що порт, на якому слухає SSH-сервер у Docker контейнері, відкритий і мапується на відповідний порт хост-системи.
```shell
docker stop $(docker ps -q)
```
